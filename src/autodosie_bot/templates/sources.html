Состав досье

Из каких блоков собирается проверка

Сценарий не одинаков для всех запросов, поэтому источники сгруппированы по реальному смыслу, а не по названию сервисов.

01

VIN: официальный поток ГИБДД

  • Сервис сразу ведет в официальный маршрут ГИБДД по VIN.
  • После капчи собираются история регистрации, розыск, ограничения, техосмотр и ДТП.
  • Результат возвращается единым читаемым отчетом внутри сайта.
02

Госномер: публичные следы

  • Автоматический поиск карточки номера и фотографий через Nomerogram.
  • Если номер уже публиковался, отчет показывает карточку, описание и найденные фото.
  • Для ограничений, розыска и регистраций сервис переводит к проверке по VIN.
03

Следующий ручной шаг

  • НСИС для проверки ОСАГО по VIN или госномеру.
  • ФНП для проверки залогов, когда машина уже идентифицирована.
  • Если VIN не найден, самый надежный путь все еще проходит через документы продавца.

Принципы интерфейса

Почему новый дизайн работает спокойнее и точнее

Вместо темной витрины здесь редакционный формат: меньше шума, больше доверия к фактам и к следующему шагу проверки.

Ритм Сначала маршрут, потом детали

Интерфейс сначала объясняет логику проверки и только затем раскладывает найденные записи по разделам.

Честность Ограничения источников не прячутся

Если нужен VIN, ручной ввод капчи или внешняя проверка, это обозначено прямо внутри сценария.

Фокус Отчет похож на рабочее досье

Ключевые сигналы находятся сверху, а вся дополнительная расшифровка уходит в структурированные карточки ниже.
